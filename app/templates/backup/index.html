Backup Management

Create Backup

Create a backup of all your data including database records and uploaded files. The backup will be downloaded as a ZIP file that can be restored later.

Uncheck to create a database-only backup (smaller file size)

Restore Backup

Restore data from a previously created backup file. This will add the backup data to your current database.

Select a backup ZIP file to restore

⚠️ Warning: This will delete all current data!

Important Information

About Backups

  • What's included: Database records (universities, courses, students, enrollments, exams, grades) and optionally uploaded files
  • File format: ZIP archive containing JSON data and files
  • Compatibility: Backups can be restored on any Dozentenmanager instance with the same version

Best Practices

  • Create regular backups before making major changes
  • Store backup files in a safe location outside the application
  • Test your backups by restoring them on a test instance
  • Be careful with the "clear existing data" option - it cannot be undone!

Restore Behavior

  • Without "clear existing": Backup data is added to current data (may cause duplicates)
  • With "clear existing": All current data is deleted first, then backup is restored
